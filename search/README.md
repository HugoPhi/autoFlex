# 搜索空间实验 (Search Space Exploration)

本实验以FlexOS为例，通过系统化地探索多维配置空间，揭示性能隔离与系统灵活性之间的权衡。核心贡献包括：配置偏序关系的数学建模、假设驱动的探索策略、以及性能异常的自动检测。

## 核心图表与论文式解说

### 1. **图 8a - 配置偏序关系图 (Configuration Poset DAG)**

**文件**: `svg/fig08_plot.svg` | `png/fig08_plot.png`

**论文式解说**：
本图展示了96个配置点在特性空间中的偏序关系，通过有向无环图(DAG)编码了"更强隔离"的传递关系。图中：
- **节点级别**：从上至下代表不同的隔离强度等级，越向下隔离约束越多
- **边的含义**：若配置A指向配置B，表示A中的所有功能隔离都包含在B中（"更弱"关系）  

**图例说明**：
- **白色节点**：初始或基础配置
- **浅灰色到深灰色节点**：根据硬化级别的渐进灰度，表示隔离强度从弱到强 (0=无隔离，1=最强隔离)
- **黑色边**：DAG中的偏序关系，指向更强隔离的配置
- **节点标签**：展示配置ID和关键性能指标（REQ_mean 等）

- **应用**：该DAG用于设计有效的配置搜索策略，避免无关的配置组合

**关键指标**：96个顶点，288条边，DAG深度=5层

### 2. **图 8b - 假设验证散点图 (Hypothesis Validation Scatter)**

**文件**: 
- `png/hypothesis_a2b_scatter_nginx_req.png` (Nginx 请求)
- `png/hypothesis_a2b_scatter_redis_get.png` (Redis GET)
- `png/hypothesis_a2b_scatter_redis_set.png` (Redis SET)

**论文式解说**：
这三个散点图验证了核心假设：**"性能隔离的强化应该导致性能下降（单调性）"**。

每个散点代表一对相邻的配置(A, B)，其中B比A的隔离更强。坐标轴为两配置的性能值(单位：时钟周期)。
- **绿色点（在直线 y=x 下方或直线上）**：符合假设，共255/259/263个点
- **橙色点（在直线上方）**：违反假设的异常，共33/29/25个点
- **最大间隔**：Nginx REQ的最大性能滑落达1243个周期（+4.6%开销）

**发现**：约88%的配置对遵循单调性假设，但离群点指示存在"隐藏的成本"，如缓存失效或上下文切换

### 3. **图 8c - Nginx 搜索路径 (Search Path Traversal)**

**文件**: `svg/nginx_search_path.svg` | `png/nginx_search_path.png`

**论文式解说**：
此图展示了高效配置搜索算法的执行轨迹（23步搜索路径）。算法从初始配置出发，通过遍历偏序关系图，在满足性能阈值(45,000周期)的所有有效配置中找到最强隔离。

**图例说明**：
- **蓝色节点（加粗边框）**：搜索路径中访问过的配置节点
- **绿色节点（浅绿色填充）**：可行的配置（满足性能约束 ≤ 45,000 周期）
- **红色双圆节点（加粗边框）**：最终答案，即满足约束的最后一个配置 (最强隔离点)
- **灰度节点**：未访问的其他配置，灰度深度表示其隔离强度
- **灰色边**：配置间的偏序关系，表示探索时的可能转移方向
- **节点提示信息**：悬停可看到该配置的详细信息 (配置ID、硬化级别、性能值)

搜索过程流程：
- **起点**：图中的初始蓝色节点（弱隔离）
- **每一步**：算法沿着边向下探索，标记已访问节点为蓝色
- **探索决策**：当发现绿色节点时，记录为可行解；继续搜索以找到最强隔离
- **终点**：最终标记为红色双圆的节点，代表找到的最优解

**性能启示**：通过DAG指导的搜索相比穷举遍历（96!种可能）效率提升多个数量级

### 4. **假设违反统计报告 (Anomaly & Violation Statistics)**

**文件**: `result/hypothesis_all_violations.csv` | `result/hypothesis_violate_report.md`

**论文式解说**：
详细的异常分析数据集，包含所有87处违反单调性假设的配置对：

| 应用 | 违规数 | 最大滑落 | Epsilon比例 |
|------|--------|---------|-----------|
| Nginx REQ | 33 | 1243周期 | 3.60% |
| Redis GET | 29 | 13,729周期 | 7.87% |
| Redis SET | 25 | 8,961周期 | 6.90% |

**论文解读**：
- Redis的异常比例更高（>6%），暗示隔离机制对内存操作的影响更大
- 最大离群值达到7.87%，超过大多数系统设计的容限（通常5%）
- 这些数据支持论文关于"性能隔离的非线性成本"的核心观点

## 数据与生成方法

### 配置空间定义
- **基础特性**：7个二进制隔离特性（如EPT、KPTI、AMD隔离等产品）
- **配置范围**：$2^7 = 128$个理论配置，采样96个
- **参数空间**：由 `data/nginx_config_map.csv` 和 `data/redis_config_map.csv` 定义

### 生成脚本
```bash
cd search/
PNG_DPI=300 bash run_debug_generate.sh
```

### 输出产物
- 所有图表生成到 `result/{svg,png}/` 目录
- 中间产物（CSV、DOT）用于验证和可视化

## 查看与应用

### 论文中的应用
1. **动机部分**：使用图8a说明配置空间的复杂性
2. **方法部分**：用图8b验证假设，用图8c阐明搜索策略
3. **结果部分**：引用统计数据支持性能分析

### 高分辨率输出
- **打印/PDF**：使用 PNG（DPI 300，>2MB）
- **编辑修改**：使用 SVG（可在Adobe/Inkscape中调整颜色、字体）
- **演讲展示**：PNG（1080p 或 4K 分辨率友好）
